椭圆曲线加密算法

01 16 2021

在之前的文章《非对称加密与大数分解》介绍过RSA加密，它是基于数学上的整数分解问题而构造的一个加密算法。当然它也不是唯一一个基于数学难题的非对称加密算法。在那篇文章里，还提到过椭圆曲线问题。那么在这篇文章中，我们就来看一个基于椭圆曲线问题的加密算法吧。

注：本文很大程度上参考了”Elliptic Curve Cryptography: a gentle introduction“系列文章，它的作者还编写了一个小工具，让读者能直观的了解椭圆曲线上的运算，非常推荐阅读。

椭圆曲线

椭圆曲线加密算法（Elliptic Curve Cryptography，简称ECC）是RSA之外另一个非对称加密算法。同样是基于数学问题，但椭圆曲线问题相比于整数分解问题却要复杂不少，不过它们也有相似之处。当然在了解椭圆曲线问题之前，我们需要先了解一下什么是椭圆曲线，以及椭圆曲线上面的运算。

椭圆曲线是一个由如下方程所描述的曲线：y2=x3+ax+by^2=x^3+ax+by2=x3+ax+b，其中aaa和bbb满足4a3+27b2≠04a^3+27b^2 \not = 04a3+27b2=0。它是一类三次平面曲线，xxx的最高次数是3。以下是摘自Wolfram MathWorld网站上椭圆曲线页面的一些图例

在图里可以看到，椭圆曲线都是关于xxx轴对称的。并且它可以是一条单独的曲线，也可能是两条不连通的曲线。

椭圆曲线上的加法

知道了椭圆曲线之后，就要了解一下椭圆曲线上的加法运算了。人们在椭圆曲线上定义了一种运算，虽然叫做加法，但是跟一般数字的加法不太相同。这里要再补充一个概念，为了让椭圆曲线上的加法运算闭合，人们需要一个额外的点。这个点位于无穷远处，叫做000点，也就是零点的意思。但是这里不要把它跟坐标原点相混淆，可以把它想象成坐标系外的一个特殊的点。椭圆曲线的加法运算就是在这个{(x,y)∈R2∣y2=x3+ax+b,4a3+27b2≠0}∪{0}\{ (x,y) \in \R^2 | y^2=x^3+ax+b, 4a^3+27b^2 \not = 0 \} \cup \{0\}{(x,y)∈R2∣y2=x3+ax+b,4a3+27b2=0}∪{0}的点集上进行的。

在我们定义好了加法运算的作用范围之后，就可以开始定义加法运算本身了。椭圆曲线上的加法的规则如下：

  • 对于椭圆曲线上任意一点PPP，有P+0=PP+0=PP+0=P。它表示一个点加上000点仍然等于它自己。
  • 对于椭圆曲线上任意一点PPP，我们把跟它相对xxx轴对称的点，记作−P-P−P，并且有P+(−P)=0P+(-P)=0P+(−P)=0。
  • 对于椭圆曲线上共线的三个点P,Q,RP, Q, RP,Q,R，有P+Q+R=0P+Q+R=0P+Q+R=0。

对于第三条，可以参考下面的示意图，如图所示的三个点P,Q,RP, Q, RP,Q,R的和就是000。

基于以上几点，椭圆曲线的加法便可以通过这样的方式来计算：如果我们有两个点P,QP, QP,Q，想要找到P+QP+QP+Q，那么就只需要做一条直线穿过PPP和QQQ，记它与曲线相交的另一点为RRR。那么这个点关于xxx轴对称的点−R-R−R，就是我们要找的P+QP+QP+Q的答案。如下图：

但是又如果PPP点是椭圆曲线与xxx轴的交点呢？这种情况下这条切线与椭圆曲线只有一个交点，此时P+PP+PP+P又是多少呢？也简单，就是那个位于无穷远的000点。

到此椭圆曲线上的加法也就定义好了。

最后，在定义好了加法之后，我们还可以继续定义整数和点的乘法（又叫做标量乘法）了。这里定义n×P=P+P+...+P⏟n个n×P = \underbrace{P+P+...+P}_{\text{n个}}n×P=n个P+P+...+P​​。计算方法也不难，就是一个一个加起来。这里随着nnn的增大，P+P+...P+P+...P+P+...也可以一直算下去。

离散有限域上的椭圆曲线

前面说的加法和乘法是在整个实数域上进行的。然而加密算法中所需要计算，是在一个缩小的范围内进行的。这里方程y2=x3+ax+by^2=x^3+ax+by2=x3+ax+b中的aaa和bbb不再是实数，而是整数。参与运算的也不再是任意的点，而是拥有整数坐标值的点。并且还需要一个质数ppp作为模数，让这些点的横竖坐标对这个模数ppp求余。这样得到的一个{(x,y)∈𝔽2∣y2≡x3+ax+b(modp),4a3+27b2≢0(modp)}∪{0}\{ (x,y) \in 𝔽^2 | y^2 \equiv x^3+ax+b \pmod p, 4a^3+27b^2 \not \equiv 0 \pmod p\} \cup \{0\}{(x,y)∈F2∣y2≡x3+ax+b(modp),4a3+27b2≡0(modp)}∪{0}的点集，就叫做离散有限域上的椭圆曲线。

下面是从”Elliptic Curve Cryptography: a gentle introduction“系列文章中摘过来的图例。它们都是基于y2≡x3−7x+10(modp)y^2 \equiv x^3-7x+10 \pmod py2≡x3−7x+10(modp)的，并分别对应着p=19,97,127,487p=19,97,127,487p=19,97,127,487。图中的点都是拥有整数坐标的点，并且可以发现，它们的图像都是关于直线y=p/2y=p/2y=p/2对称的。

那么在这个离散的椭圆曲线上，我们又要如何进行加法呢？我们还是定义对于任意的共线的三个点P,Q,RP, Q, RP,Q,R，有P+Q+R=0P+Q+R=0P+Q+R=0。不过在有限域上的共线与一般的共线略有不同。下图是一个在椭圆曲线y2≡x3−x+3(mod127)y^2 \equiv x^3-x+3 \pmod {127}y2≡x3−x+3(mod127)上的示例，其中P=(16,20)P=(16,20)P=(16,20)，Q=(41,120)Q=(41,120)Q=(41,120)。可以看到离散有限域上的直线，是在区域内循环的。

至于一个离散椭圆曲线里有多少个点呢？显然一个一个找并不现实，尤其是当ppp很大的时候。不过这里有一个Schoof's算法，可以快速算出这个数。另外离散椭圆曲线里点的数量，又叫做这个离散椭圆曲线的阶(order)。

离散椭圆曲线上的标量乘法

加法定义好了，我们同样也可以定义离散椭圆曲线上的标量乘法了，只要让一般椭圆曲线上的标量乘法的结果对ppp求余就好。并且计算后会发现，离散椭圆曲线上的标量乘法，有一个很有趣的性质。拿y2≡x3+2x+3(mod97)y^2 \equiv x^3+2x+3 \pmod {97}y2≡x3+2x+3(mod97)举个例子，对于点P=(3,6)P=(3,6)P=(3,6)，它的倍数如下：

  • 0P=00P=00P=0
  • 1P=(3,6)1P=(3,6)1P=(3,6)
  • 2P=(80,10)2P=(80,10)2P=(80,10)
  • 3P=(80,87)3P=(80,87)3P=(80,87)
  • 4P=(3,91)4P=(3,91)4P=(3,91)
  • 5P=05P=05P=0
  • 6P=(3,6)6P=(3,6)6P=(3,6)
  • ...

可以观察到，PPP的倍数是成周期变化的，周期是555。写成式子，有kP=(kmod  5)PkP=(k \mod 5)PkP=(kmod5)P，比如6P=(6mod  5)P=1P6P=(6 \mod 5)P=1P6P=(6mod5)P=1P。这样实际上，0P,1P,2P,3P0P,1P,2P,3P0P,1P,2P,3P和4P4P4P这五个点，就涵盖了kPkPkP的取值范围。也可以说，无论kkk是多少，kPkPkP也只能是这五个点之中的一个。

还有一个性质之前没讲到，就是椭圆曲线的标量乘法是满足结合律的，即mP+nP=(m+n)PmP+nP = (m+n)PmP+nP=(m+n)P。当然这一点也是显而易见，因为mP+nP=P+...+P⏟m个+P+...+P⏟n个=(m+n)PmP+nP = \underbrace{P+...+P}_{\text{m个}} + \underbrace{P+...+P}_{\text{n个}} =(m+n)PmP+nP=m个P+...+P​​+n个P+...+P​​=(m+n)P。

最后，总结一下上面的性质就能发现，实际上PPP的倍数组成了一个离散椭圆曲线的循环子群1。由于这个循环子群是由PPP算出来的，我们便称PPP为这个循环子群的基点（base point）。另外这个循环子群中点的个数叫做这个子群的阶，也可以简称它是PPP的阶。我们将椭圆曲线的阶记为NNN，将PPP的阶记为nnn，那么它们还具有如下两个性质：

  • nP=0nP=0nP=0
  • NNN是nnn的整数倍。

对于第二点，我们把NNN除以nnn的结果记为hhh，它又叫做这个循环子群的cofactor。我并没能找到它对应的中文翻译，就叫它hhh好了。

离散椭圆曲线上的离散对数问题

到这里终于算是将椭圆曲线的基础知识介绍完了！终于可以开始介绍椭圆曲线加密本身了。

椭圆曲线加密算法本身，是基于椭圆曲线上的离散对数问题而建立的。这个问题听名字好像很复杂，但实际上描述起来却很简单。对于椭圆曲线上的一个基点GGG，我们可以很容易的算出它的任意倍数H=kGH=kGH=kG。但是反过来，在知道GGG和HHH的情况下，想要算出这个kkk却非常难！尤其是当椭圆曲线的参数aaa和bbb，以及模数ppp都非常大的时候。而正是这个难度的不对称，成为了椭圆曲线加密算法的基础。

椭圆曲线的加密算法的一个主要应用就是椭圆曲线迪菲-赫尔曼秘钥交换（Elliptic Curve Diffie–Hellman key Exchange，简称ECDHE），通过这个方法，通信双方可以交换一个共有的密钥，并在之后使用这个密钥加密他们两者之间的通信内容。

具体操作如下，假如Alice和Bob两人要进行通信。他们使用一个事先约定好的椭圆曲线，并找出一个合适的基点GGG，使得这个基点对应的阶nnn是一个质数。之后他们就可以开始进行密钥交换。第一步，他们需要先分别随机找到两个数字dAd_AdA​和dBd_BdB​，并分别算出HA=dAGH_A=d_A GHA​=dA​G以及HB=dBGH_B=d_B GHB​=dB​G。其中dAd_AdA​和dBd_BdB​分别是Alice和Bob的私钥，HAH_AHA​和HBH_BHB​是对应的公钥。第二步，Alice把HAH_AHA​告诉Bob，Bob也将HBH_BHB​告诉Alice。最后一步，Alice计算S=dAHBS=d_A H_BS=dA​HB​，Bob计算S=dBHAS=d_B H_AS=dB​HA​，这里他们会得到同一个SSS，因为S=dAHB=dA(dBG)=dB(dAG)=dBHA=SS=d_A H_B = d_A(d_B G) = d_B(d_A G) = d_B H_A = S S=dA​HB​=dA​(dB​G)=dB​(dA​G)=dB​HA​=S。这样他们就有这么一个共有的知识SSS，可以使用它来进行对称加密。

这里最关键的一点，就算有一个第三者，截获了Alice和Bob的通信，得到了HAH_AHA​和HBH_BHB​，他也无法在不知道dAd_AdA​和dBd_BdB​的情况下，算出SSS。因为要算出SSS，就需要dAd_AdA​或dBd_BdB​，但前面说了，就算在知道HA(HB)H_A(H_B)HA​(HB​)和GGG的情况下，要算出dA(dB)d_A(d_B)dA​(dB​)也非常难。

另外为了提高安全性，通信密钥也要经常更换。更换起来也不难，只需要Alice和Bob再分别找两个其他的数字dAd_AdA​和dBd_BdB​，重新进行一遍秘钥交换就行。

目前互联网上面的传输层安全性协议（TLS）以及OpenSSL，都使用了ECDHE。你可以在OpenSSL的源代码里看到它使用的是哪些椭圆曲线。

椭圆曲线的另一个应用，是椭圆曲线数字签名算法（Elliptic Curve Digital Signature Algorithm，缩写ECDSA）。这个算法被用于比特币的加密（前文《非对称加密与大数分解》中说比特币使用的是RSA算法，其实是错误的，这里做更正）。

椭圆曲线的选择

最后这里还涉及到一个问题，就是这个椭圆曲线要怎么找。实际使用中人们发现，并不是所有椭圆曲线都适合，有些椭圆曲线相比之下会弱一些。比如当p=hnp=hnp=hn的时候，就有一些攻击方法能在多项式时间内算出H=kPH=kPH=kP中的kkk2。为此我们就需要先找到一些适合的椭圆曲线，才能进行椭圆曲线加密。

有一种方法是随机生成椭圆曲线，再计算它的阶是多少，有没有合适的基点等等。随机生成的方法也不算麻烦，可以使用某些随机数生成器（Random Number Generator），生成一个随机数RRR，再计算它的哈希值H=hash(R)H = hash(R)H=hash(R)，再通过一些方法算出a=f(H)a=f(H)a=f(H)和b=g(H)b=g(H)b=g(H)作为椭圆曲线的参数。在aaa和bbb确定了之后，我们就可以使用它来进行椭圆曲线加密了。

参数aaa和bbb由随机以及求哈希值的方法计算出来，可以确保它们不是被某人精心设计过的。因为有的曲线可能相对比较弱，或者某条曲线的提出者可能在里面隐藏了后门或漏洞，能用某些方法，暗中计算出kkk来，这样的曲线就不够安全。由于要确定一条曲线是不是有漏洞不容易，不如就随机来生成椭圆曲线好了。

不过有没有可能，某个人找到了一系列有漏洞的曲线，并巧妙的设计了一个随机数生成算法，使得它随机生成的曲线都恰好是这些有漏洞的曲线呢。不好说，但确实可能。美国国家安全局（NSA）曾要求国家标准技术研究所（NIST）使用一个他们研发的随机数生成器Dual_EC_DRBG，这个随机数生成器就被认为是有漏洞的。不过虽然备受批评，直到2017年被撤销之前，Dual_EC_DRBG在七年的时间内都是NIST定义的标准安全伪随机数生成器之一。

椭圆曲线总结

椭圆曲线加密相比RSA要更加安全，它可以用更短的密钥长度达到同样的安全等级。一个使用大小为224位二进制位椭圆曲线进行的加密算法，可以跟使用2048位二进制位质数的RSA算法达到相当的安全程度。一个原因就是对于椭圆曲线的离散对数问题，我们并没有找到相对好的传统算法，而RSA可以使用普通数域筛选法GNFS。黑莓公司的子公司Certicom于1997年曾发出了一个挑战，给出了7个长度在109到359位之间的椭圆曲线。至今为止，只有109位的椭圆曲线被破解了。不过由于秀尔算法可以快速的解决离散对数问题，使得椭圆曲线加密算法仍有可能受到量子计算的威胁。
